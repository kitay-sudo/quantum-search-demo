## Что это?

**Алгоритм Гровера** - квантовый алгоритм поиска элемента в неупорядоченной базе данных.

### Ускорение
- Классический поиск: O(N) - проверка по одному
- Квантовый (Гровер): O(√N) - все состояния одновременно!

**Пример:** Поиск 1 из 1,000,000 элементов
- Классический: ~500,000 проверок
- Квантовый: ~1,000 итераций
- **Ускорение: 500x!**

## Как работает алгоритм?

### Два шага (повторить ~√N раз):

1. **Oracle (Оракул)** - помечает правильный ответ (меняет знак амплитуды)
2. **Diffusion (Диффузия)** - усиливает вероятность (отражение от среднего)

### Квантовое преимущество

**Суперпозиция:**
- N кубитов = 2^N состояний ОДНОВРЕМЕННО
- 20 кубитов = 1,048,576 состояний сразу!

**Интерференция:**
- Амплитуды складываются и вычитаются
- Неправильные ответы гасят друг друга
- Правильный ответ усиливается

## FAQ

**Q: Это настоящий квантовый компьютер?**
A: Нет, это симуляция на обычном компьютере.

**Q: Почему амплитуды отрицательные?**
A: Квантовая фаза! Oracle меняет знак, создавая интерференцию.

**Q: Каждый раз разная цель?**
A: Да, случайная. Зафиксировать: `target = 7` вместо `random.randrange(N)`
